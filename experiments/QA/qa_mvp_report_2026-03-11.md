# QA-отчёт по MVP-версии DSPro VKR

Дата: 11 марта 2026 года

Статус документа:
- historical QA slice;
- фиксирует ранний, более узкий QA-прогон до полного file-by-file,
  architecture и documentation audit;
- не должен читаться как главный current-state QA-документ.

## Цель проверки

Подтвердить, что текущая версия проекта работоспособна как научный MVP:

- полный pipeline выполняется end-to-end;
- математический контракт модели корректен;
- физическая логика не противоречит базовым ожиданиям;
- кодовая база и тесты находятся в воспроизводимом состоянии.

Важно: текущая версия не претендует на полную астрофизическую иерархию всех подклассов звёзд. Это первая научная итерация, цель которой — проверить жизнеспособность выбранной архитектуры и понять, оправдано ли дальнейшее углубление модели.

## Проверка runtime-контракта

Проверено, что в результатах отсутствуют явные численные и контрактные нарушения:

- нет `NaN/inf` в ключевых score-полях;
- `host_posterior` лежит в диапазоне `[0, 1]`;
- `final_score` лежит в диапазоне `[0, 1]`;
- `posterior_margin >= 0`;
- обязательные поля router и priority результатов заполнены;
- логика ветвления `host / low-priority` согласована с reason codes.

## Физические sanity-check

Для MVP-постановки физическая картина выглядит правдоподобно.

### Температурная последовательность по медиане `teff_gspphot`

- `B > A > F > G > K > M`

Это соответствует ожидаемому порядку спектральных классов.

### Разделение `dwarf / evolved`

По агрегатным значениям:

- dwarfs имеют более высокий `logg`;
- evolved имеют больший радиус и меньший `logg`.

Это согласуется с базовой физической интерпретацией текущего router-слоя.

## Математические sanity-check

Проверены production-артефакты router и host-модели:

- `effective_cov` положительно определены;
- `log_det_cov` конечны;
- критических проблем со shrinkage и near-singular covariance не выявлено;
- posterior/contrastive-контур численно устойчив.

## Проверка качества кода

- `ruff` — без ошибок

## Аудит тестов

Сильные стороны текущего набора тестов:

- хорошо покрыт contract router artifact и scoring;
- хорошо покрыт contrastive host artifact и scoring;
- есть регрессии на backward compatibility фасадов;
- есть проверки persist payload contract;
- есть DB-backed интеграционный тест на временной схеме Postgres для `run_pipeline(..., persist=True)`;
- есть DB-backed интеграционные тесты для `input_layer.py` и `lab.input_dataset_registry`;
- есть проверки EDA artifact contracts.

Ограничения текущего набора тестов:

- физические sanity-checks пока оформлены как QA-анализ, а не как автоматические тесты.

## Ограничения MVP-версии

Это не баги, а осознанные ограничения текущей итерации:

- модель не покрывает полную иерархию всех астрофизических подклассов;
- редкие подпопуляции поддерживаются на ограниченной статистике;
- отдельные хвостовые группы, например `M_evolved` или `M_LATE field`, пока имеют слабую статистическую устойчивость;
- текущая версия предназначена для проверки идеи, а не для окончательной физической модели.

## Итоговый вывод

Текущая версия проекта можно считать успешной научной MVP-итерацией.

Основной вывод QA:

- выбранная архитектура работает end-to-end;
- численно она ведёт себя устойчиво;
- физическая логика выглядит правдоподобно на уровне, достаточном для MVP ВКР;
- дальнейшее развитие модели имеет смысл.

Иными словами, текущая версия подтверждает, что гипотеза о полезности связки:

- physical Gaussian router;
- contrastive host-vs-field model;
- decision layer с коэффициентами качества

является рабочей и заслуживает дальнейшего углубления в следующих версиях программы.
