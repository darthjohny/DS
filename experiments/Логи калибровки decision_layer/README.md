# Логи калибровки decision_layer

В этой папке хранится история ручной калибровки итогового `final_score`.

## Назначение

- фиксировать параметры одной итерации;
- сохранять гипотезу, фактический результат и следующий шаг;
- быстро сравнивать разные варианты калибровки;
- держать единый формат экспериментального журнала для ВКР и отладки.

## Формат файлов

- `iteration_001.md`
- `iteration_002.md`
- `iteration_003.md`

Стандартный каркас одной итерации:

1. шапка: идентификатор, дата, статус;
2. контекст итерации;
3. что меняем;
4. формула и режимы скоринга;
5. гипотеза;
6. параметры итерации;
7. ожидаемый эффект;
8. фактический результат;
9. сводка по top-N;
10. итог;
11. следующий шаг.

Важно:

- исторические заполненные итерации могут отражать старые версии формулы;
- новые файлы создаются уже по актуальному шаблону с `host_posterior`.

## Как создать новый файл

```bash
python src/logbooks/decision_layer.py
```

После запуска появится следующий по номеру markdown-файл с готовым шаблоном в актуальном формате.
