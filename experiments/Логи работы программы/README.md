# Логи работы программы

В этой папке хранится журнал прогонов боевого pipeline.

## Назначение

- фиксировать каждый запуск оркестратора;
- сохранять контекст запуска, версии моделей и ключевые метрики;
- быстро восстанавливать, что именно запускалось и какой результат это дало;
- держать единый формат журналов для ВКР, отладки и воспроизводимости.

## Формат файлов

- `run_001.md`
- `run_002.md`
- `run_003.md`

Стандартный каркас одного файла:

1. шапка: идентификатор, дата, статус;
2. контекст запуска;
3. артефакты и режимы;
4. параметры decision layer;
5. размеры потоков;
6. ключевые метрики;
7. кандидаты top-N;
8. предупреждения и ошибки;
9. итог;
10. следующий шаг.

## Как создать новый файл

```bash
python src/logbooks/program_run.py
```

После запуска появится следующий по номеру markdown-файл с готовым шаблоном в актуальном формате.
