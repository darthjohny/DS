## Быстрый Итог

Полностью без шапок активные `.py`-файлы сейчас не найдены.

На момент первичного аудита единый структурный шаблон держался не везде:

- `src/exohost`: `229` файлов, из них `175` с ослабленным или неполным форматом;
- `tests/unit`: `148` файлов, из них `142` с ослабленным или неполным форматом.

Под “ослабленным форматом” здесь понимаются файлы, где:

- есть только короткая первая строка без структуры;
- нет блока `Этот файл отвечает только за:`;
- нет блока `Следующий слой:`.

После волны стандартизации active-слоя:

- `src/exohost`: `0` файлов без структурной шапки;
- `tests/unit`: `0` файлов без структурной шапки.

## Что Это Значит Практически

Проблема сейчас не в отсутствии комментариев вообще, а в отсутствии
последовательного стандарта.

Из-за этого:

- по проекту трудно быстро пройти глазами;
- сложно строить карту слоев для презентации;
- README позже будет трудно превратить в понятное оглавление.

## Выполненный Рабочий Пакет

В первой волне был выровнен активный `reporting`-слой и связанные
`tests/unit/reporting`, потому что:

- именно он сейчас напрямую поддерживает notebook и итоговые review;
- именно из этого слоя проще всего строить схему проектной логики;
- там уже есть часть хороших шапок, и можно быстро выровнять стиль.

После пилотной волны стандарт был распространен на весь active `src/tests`.

## Проверки После Стандартизации

После массового выравнивания шапок активный слой был проверен через:

- `ruff check src tests`
- `mypy src tests`
- `pyright src tests`
- `pytest -q tests`

Все проверки прошли успешно.
