# Стабилизация и служебная политика

Здесь лежат документы рабочего контура, которые поддерживают проект в
упорядоченном состоянии:

- cleanup-аудит и cleanup-план;
- ledger найденных проблем;
- политика notebook QA;
- политика архива исследований;
- политика и runbook регресс-тестирования.

Этот раздел отвечает на вопрос: «как мы держим проект в рабочем состоянии».

Это служебный слой проекта. Он полезен для понимания того, как поддерживается
качество и воспроизводимость, но не должен превращаться в свалку внутренних
напоминаний и частной инженерной переписки.
