## О какой группе идет речь

Верхняя группа проекта — это объекты из текущего базового прогона, для которых:

- `final_domain_state = id`
- `final_quality_state = pass`
- `priority_label = high`

На текущем базовом прогоне в эту группу входит:

- `72 113` объектов.

Именно этот слой отвечает на главный прикладной вопрос проекта:

- какие звезды сейчас выглядят наиболее перспективными для последующих
  наблюдений в рамках логики поиска объектов, похожих на host-популяцию.

## Что это за объекты по крупным классам

Верхняя группа не выглядит как случайная смесь всех классов подряд.

По крупным спектральным классам она устроена так:

- `K = 30 114`
- `F = 27 488`
- `G = 14 341`
- `M = 170`

Практический вывод:

- верхняя группа почти целиком состоит из `F/G/K` объектов;
- горячие массивные классы не доминируют;
- верхняя группа ближе к физически ожидаемому профилю звезд, похожих на
  host-популяцию, чем к “экзотическому” хвосту каталога;
- для основной прикладной задачи это и есть хороший знак: проект поднимает
  наверх не редкие горячие гиганты, а более спокойные `F/G/K`-объекты, которые
  выглядят естественными целями для поиска объектов, похожих на host-популяцию.

## Что это за объекты по подклассам

Самые заметные подклассы в верхней группе:

- `K0 = 21 283`
- `F5 = 9 059`
- `G0 = 7 229`
- `F8 = 6 983`
- `F0 = 5 352`
- `G5 = 5 128`
- `K2 = 4 392`
- `F2 = 3 746`

Это важно, потому что верхний слой:

- не схлопнулся в один единственный подкласс;
- но и не распался в полностью хаотичную смесь;
- он концентрируется вокруг физически похожих зон пространства признаков.

## Почему эти объекты поднимаются наверх

В текущем контуре верхняя группа формируется в основном двумя причинами:

- `сильное сходство с host`
- `сильное сходство с host; хорошая наблюдательная пригодность`

То есть объект поднимается наверх не потому, что он “интересный вообще”, а
потому что:

1. он похож на host-профиль по обученной host-модели;
2. для части объектов это дополнительно усиливается наблюдательной
   пригодностью.

Иными словами, верхняя группа — это результат не одной модели, а связки:

- маршрута с учетом качества данных;
- `ID`-слоя;
- оценки сходства с host-профилем;
- оценки наблюдаемости;
- итоговой логики назначения приоритета.

Интерпретация этого профиля:

- `ruwe` и `parallax_over_error` говорят о хорошей астрометрии;
- `phot_g_mean_mag` показывает, что группа в среднем достаточно яркая;
- `teff_gspphot` уводит верхний слой не в горячие редкие звезды, а в более
  спокойный `late-F / G / K` диапазон.

## Что это значит для прикладной задачи

В контексте ВКР эту группу можно интерпретировать так:

- проект не просто разметил звезды по классам;
- он выделил компактный и интерпретируемый слой целей, которые:
  - похожи на известные host-профили;
  - обладают хорошим качеством данных;
  - выглядят удобными для последующих наблюдений.

То есть верхняя группа — это рабочий список целей:

- не окончательный научный вердикт;
- не каталог подтвержденных экзопланет;
- а список лучших кандидатов для следующего наблюдательного шага.

## Как эту группу правильно описывать в тексте ВКР

Корректная формулировка:

> В результате работы был сформирован верхний список из `72 113`
> объектов, которые по совокупности признаков демонстрируют наибольшее сходство
> с профилем известных звезд-хозяев и одновременно обладают хорошими
> характеристиками наблюдаемости. Эта группа не трактуется как набор
> подтвержденных планетных систем, а рассматривается как практически полезный
> слой кандидатов для дальнейших наблюдений.

Некорректная формулировка:

- “модель нашла `72 113` звезд с планетами”;
- “верхняя группа доказывает наличие экзопланет”;
- “все объекты этой группы одинаково перспективны”.

## Почему это важно для ВКР

С точки зрения ВКР именно этот слой является главным прикладным выходом работы,
потому что он переводит задачу из абстрактной классификации в практику
наблюдений.

Именно здесь видно, что:

- Data Science в проекте используется не ради одной метрики;
- контур обработки действительно помогает приоритизировать наблюдательные цели;
- результат можно интерпретировать физически и инженерно, а не только
  статистически;
- а отдельные сложности вроде `O/B` не отменяют основной прикладной полезности
  этого слоя.

## Вывод

Верхняя группа проекта — это не побочный продукт ранжирования, а главный
прикладной слой работы.

Он:

- физически выглядит правдоподобно;
- доминируется `F/G/K` объектами;
- держится на очень сильном сходстве с host-популяцией;
- усиливается наблюдательной пригодностью;
- может быть честно интерпретирован как список лучших целей для дальнейшего
  наблюдения в рамках текущего контура.
