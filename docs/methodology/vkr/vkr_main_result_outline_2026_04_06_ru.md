## Зачем Нужен Этот Документ

Этот документ фиксирует не все детали проекта, а главный прикладной результат,
который потом должен войти в текст ВКР.

Его задача:

- отделить основной результат от служебной инженерной работы;
- дать короткую и честную формулировку того, что именно получилось;
- зафиксировать, как правильно трактовать верхний итоговый shortlist.

## Что Именно Построено

В проекте построен воспроизводимый pipeline, который:

1. читает и нормализует астрономические данные;
2. использует внешние спектральные метки для классификации звезд по классам и
   подклассам;
3. оценивает качество и надежность входных данных;
4. отделяет рабочую область от сомнительных и внешних объектов;
5. оценивает сходство звезды с профилем известных host-объектов;
6. формирует итоговый приоритет наблюдений.

На уровне прикладной задачи pipeline отвечает на два вопроса:

1. к какому спектральному классу и подклассу ближе объект;
2. насколько этот объект интересен как цель для follow-up наблюдений.

## Что Считаем Главным Результатом Работы

Главный результат текущей версии проекта:

- построен стабильный shortlist верхнего приоритета;
- в этот shortlist входит `72 113` объектов;
- shortlist получен на current active baseline:
  - [hierarchical_final_decision_2026_04_05_123111_055017](/Users/evgeniikuznetsov/Desktop/dspro-vkr/artifacts/decisions/hierarchical_final_decision_2026_04_05_123111_055017)

Практический смысл этого результата:

- это не список “обнаруженных экзопланет”;
- это не список “подтвержденных звезд-хозяев”;
- это слой объектов, которые по совокупности признаков сильнее всего похожи на
  host-like кандидаты и лучше всего подходят для дальнейших наблюдений.

Именно это и является центральным прикладным результатом ВКР.

Важно:

- основная прикладная задача работы состоит именно в приоритизации наблюдений и
  выделении host-like кандидатов;
- классификация по спектральным классам и подклассам нужна здесь как важный
  supporting layer, который делает shortlist физически осмысленным;
- глубокая детализация по редким boundary-случаям не является центральным
  критерием успеха основной прикладной задачи.

## Как Правильно Трактовать Эти `72 113` Объектов

Правильная трактовка верхнего shortlist:

- объекты прошли quality-aware routing;
- они дошли до `ID`-слоя pipeline;
- для них высок `host_similarity_score`;
- для значимой части из них дополнительно хороша наблюдательная пригодность;
- именно поэтому они попали в `priority = high`.

Неправильная трактовка:

- “модель доказала наличие планет”;
- “это уже подтвержденные системы”;
- “все объекты наверху одинаково перспективны научно”.

То есть верхняя группа — это не финальный научный вердикт, а качественно
отобранный и интерпретируемый слой целей для follow-up.

## Почему Этот Результат Выглядит Осмысленным

Верхний shortlist не похож на случайную смесь объектов.

По текущему active baseline он:

- состоит в основном из `F/G/K` звезд:
  - `K = 30 114`
  - `F = 27 488`
  - `G = 14 341`
  - `M = 170`
- имеет очень высокий медианный `host_similarity_score`:
  - `0.997999`
- имеет высокий медианный `priority_score`:
  - `0.877863`
- опирается на хорошие наблюдательные и астрометрические признаки.

Это важно, потому что итоговая верхняя группа:

- физически правдоподобна;
- согласуется с host-like логикой проекта;
- не выглядит как побочный продукт шумного ranking без контроля качества.
- и, что особенно важно для основной прикладной задачи, не уходит в редкий
  горячий хвост вроде `O/B`, а концентрируется в области спокойных `F/G/K`
  объектов, которые и выглядят более естественными кандидатами для follow-up.

## Почему Результату Можно Доверять

Текущий основной результат держится не на одном удачном прогоне.

Для него уже есть:

- active baseline run;
- отдельный review по верхнему shortlist;
- tuned policy для `quality_gate` и `priority`;
- validation run после введения regression-layer:
  - [hierarchical_final_decision_2026_04_06_095722_391062](/Users/evgeniikuznetsov/Desktop/dspro-vkr/artifacts/decisions/hierarchical_final_decision_2026_04_06_095722_391062)

Validation run показал:

- полное совпадение с active baseline по:
  - `decision_input.csv`
  - `final_decision.csv`
  - `priority_input.csv`
  - `priority_ranking.csv`

Значит текущий главный результат:

- воспроизводим;
- не зависит от случайной ручной настройки;
- выдерживает проверку после введения регресс-слоя.

## Короткая Формулировка Для Будущего Текста ВКР

Работа показала, что на основе параметров звезд из Gaia DR3 и внешнего
host-контекста можно построить воспроизводимый pipeline, который не только
классифицирует звезды по спектральным классам и подклассам, но и формирует
интерпретируемый shortlist наиболее перспективных целей для последующих
наблюдений.

В текущей версии проекта такой shortlist включает `72 113` host-like объектов
верхнего приоритета. Этот результат не трактуется как прямое обнаружение
экзопланет, а рассматривается как practically useful слой кандидатов для
дальнейшего follow-up анализа.

При этом основная прикладная задача проекта уже решена: pipeline устойчиво
выделяет верхний слой host-like целей для наблюдений. Более тонкая и научно
нагруженная задача глубокой подклассовой детализации остается отдельным
направлением дальнейшей доработки.
