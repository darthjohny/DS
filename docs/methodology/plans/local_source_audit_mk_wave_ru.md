# План аудита локальных источников перед MK-волной

Дата фиксации: `2026-03-27`

## Зачем понадобился этот этап

Перед запуском новой MK-волны нужно было понять, какие локальные таблицы уже
годятся для работы, а какие не покрывают задачу по меткам и признакам.

## Цель

Зафиксировать, что уже есть в локальной базе для новой волны, а где без
внешнего источника меток обойтись не получится.

## Что проверялось

В этом аудите сравнивались:

- локальные таблицы для обучения и обзора;
- наличие классов, подклассов и этапов эволюции;
- покрытие основных физических и качественных признаков;
- пригодность таблиц для крупного класса, подкласса и дополнительных задач.

## Главный вывод

Аудит показал, что локальной базы уже достаточно для:

- крупных спектральных классов;
- слоев качества;
- части вспомогательных задач;

но недостаточно для полноценной новой волны по подклассам без внешнего
источника меток.

## Итог

Этот план зафиксировал важное решение: новый слой по подклассам должен
строиться не только на локальной базе, но и на внешнем спектральном источнике с
последующим сопоставлением в Gaia.
