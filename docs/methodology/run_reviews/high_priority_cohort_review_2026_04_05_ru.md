## Зачем нужен этот разбор

Верхняя группа `priority = high` сейчас является главным слоем отбора
проекта. Именно она отвечает на практический вопрос:

- какие объекты сильнее всего похожи на host-кандидатов;
- какие из них лучше всего подходят для дальнейших наблюдений;
- как выглядит этот список по классам, подклассам и физическим параметрам.

Важно:

- это не подтвержденные планетные системы;
- это верхний список целей для последующих наблюдений в рамках текущего
  контура.

## Почему они оказались вверху

Причины приоритета для всей верхней группы:

- `сильное сходство с host = 51 945` (`72.03%`)
- `сильное сходство с host; хорошая наблюдательная пригодность = 20 168` (`27.97%`)

Это значит, что верхняя группа практически целиком собирается двумя
механизмами:

1. объекты очень похожи на host-профиль;
2. у существенной части сверху к этому добавляется еще и сильная
   наблюдательная пригодность.

## Вывод

На текущем базовом прогоне верхняя приоритетная группа выглядит физически
правдоподобно и инженерно объяснимо:

1. это не случайная смесь по всем классам подряд;
2. верхняя группа доминируется `F/G/K` объектами;
3. верхнюю группу в основном формирует очень сильное сходство с host-профилем;
4. у самых верхних объектов дополнительно сильная наблюдательная пригодность и
   очень хорошая астометрия.

Поэтому именно этот слой сейчас можно считать главным прикладным результатом
проекта:

- не “мы нашли подтвержденные планеты”;
- а “мы построили воспроизводимый список наиболее интересных
  похожих на host
  объектов для дальнейшей проверки и последующих наблюдений”.
