# План разбора горячего подмножества O-класса

## Зачем проводился этот шаг

Этот документ сохраняет исходный план следующего шага после первого разбора
редкого хвоста `O`.

На том этапе нужно было:

- не трогать coarse-модель вслепую;
- выделить внутри true `O` только горячее подмножество, согласованное с
  физикой `O/B`;
- отделить конфликт между меткой и физическими признаками от настоящей ошибки
  редкого класса.

## Почему потребовалось горячее подмножество

Первый разбор по всему `O`-пулу показал:

- `quality_gate` режет значимую часть `O`;
- surviving `O` почти не переводится в `O`;
- часть surviving `O` уже физически не похожа на горячие звезды.

Поэтому следующим логичным шагом был разбор только тех строк, которые хотя бы
минимально согласуются с горячей физикой `O/B`.

## Базовый критерий этого шага

В качестве рабочего порога на первом проходе использовалось условие:

- `teff_gspphot >= 10000 K`

Идея была простой: отсечь явный холодный хвост и оставить минимум, похожий на
горячие `O/B`-объекты.

## Что должен был показать разбор

На этом шаге требовалось:

1. оценить размер горячего подмножества внутри true `O`;
2. посмотреть, сколько строк из него проходит `quality_state = 'pass'`;
3. проверить, исчезает ли после этого системный увод в `F/G/K/M`;
4. понять, остается ли проблема именно в маршрутизации `O -> B`.

## Почему документ сохранен в архиве

Этот план важен как промежуточная исследовательская постановка. Позднее стало
ясно, что после выделения горячего подмножества вопрос сужается до более
точной границы `O/B`, а не до всего класса `O`.
