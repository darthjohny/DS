# Domain Glossary V2

## Основные сущности

- `quality_state` — состояние качества записи и пригодности объекта к дальнейшему анализу.
- `ood_state` — признак выхода объекта за область доверия модели.
- `evolution_stage` — стадия эволюции звезды.
- `luminosity_class` — класс светимости в схеме MK.
- `luminosity_group` — укрупненная группа светимости для первой волны моделей.
- `spectral_class` — спектральный класс верхнего уровня.
- `spectral_subclass` — подкласс внутри спектрального класса.
- `peculiarity_suffix` — суффиксы спектральных особенностей, не входящие в первую волну target-меток.
- `host_similarity_score` — близость объекта к профилю известных host-звезд.
- `observability_score` — оценка пригодности объекта к последующим наблюдениям.
- `priority_score` — итоговый численный приоритет.
- `priority_label` — человекочитаемая категория итогового приоритета.

## Принцип разделения

Эти сущности нельзя смешивать в одном строковом поле.

Правильный подход:

- класс;
- подкласс;
- класс светимости;
- стадия эволюции или ее укрупнение;
- качество;
- итоговый приоритет

должны храниться и обрабатываться раздельно.
